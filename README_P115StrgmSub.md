## 🔌 必须搭配的插件

### ✅ P115StrmHelper（115网盘订阅助手）

本插件的**网盘追更功能**必须配合 `P115StrmHelper` 插件使用：

- **订阅转存流程：** 插件搜索到 115 网盘分享链接 → 转存到指定目录 → **P115StrmHelper 监控整理记录**，自动生成 Emby/Jellyfin 可识别的 strm 文件
- **文件匹配：** P115StrmHelper 通过「监控整理记录」方式，将转存后的文件匹配到正确的媒体目录，生成 strm
- **双向依赖：** 没有 P115StrmHelper，网盘订阅只能转存文件，无法生成媒体库识别的 strm

> 建议两个插件同时安装，P115StrmHelper 配置好 115 Cookie 和整理规则即可。

### 可选：联动删除 daemon（sync_subscribe_media.py）

配套的后台守护进程，实现 "删 strm → 自动清 115 回收站" 的端到端联动：

```
手动/洗版删除 strm → inotify 触发 → CD2 API 删除 115 文件（回收站）
                                    → 同步清理缩略图（th）
                                    → 更新订阅进度
```

> 如需此功能，请向维护者索取部署脚本。

## ⏰ 屏蔽态 vs 开放态（核心创新设计）

本插件**最核心的改进**，比原版「可配置窗口」机制更简洁直观。

### 原版问题

原版使用「可配置窗口」机制，多个时间配置互相作用，逻辑复杂，不支持跨天，用户很难理解。

### 魔改版设计

一天只有**两个时段**：

```
       屏蔽态                         开放态
┌──────────────────┴──────────────────┬──────────────────┴──────┐
00:00                             17:30                     23:59
```

| 时段 | 状态 | 订阅行为 |
|------|------|---------|
| 00:00 → 17:30 | **屏蔽态** 🔴 | 所有订阅 `sites=[-1]`（仅 115 网盘），不连接任何 PT 站点 |
| 17:30 → 23:59 | **开放态** 🟢 | 恢复订阅的原始站点选择，PT 正常 RSS / 选种 / 下载 |

### 优势

- **屏蔽态不碰 PT 站点** — 适合 PT 考核期/站点维护期，完全不被 PT 影响
- **开放态自动恢复** — 到点自动切回正常状态，PT 和网盘双轨运行
- **支持跨天** — 配置 `unblock_end_time` 可以跨天（原版做不到）
- **自动保存站点快照** — 屏蔽态时自动保存原始 `sites` 配置，开放态零手动操作恢复

### 配置项

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `unblock_start_time` | `17:30` | 开放态开始时间（切换到全部站点） |
| `unblock_end_time` | `23:59` | 开放态结束时间（切回屏蔽态） |

### 手动 PT 搜索（屏蔽态例外）

屏蔽态下如需执行 PT 订阅搜索，MP 微信/TG 输入命令：

```
/pt_sub_search
```

插件会自动：
1. 临时恢复订阅的原始 sites（突破 `[-1]` 限制）
2. 执行 PT RSS / 搜索
3. 60 秒余量窗口后自动锁回屏蔽态

## 🔄 洗版升级逻辑

插件支持**双轨洗版**——网盘洗版和 PT 洗版各有独立的评分拦截机制。

### 网盘洗版（插件内部驱动）

```
① 定时扫描本地 strm → 读取评分 + 查询文件体积
         ↓
② 搜索 115 新资源 → MP 规则组打分
   （综合评分 = 体积×0.75 + 规则因子×0.25，可切换纯体积方案）
         ↓
③ 新评分 - 旧评分 ≥ 阈值（默认 25 分）？
         ↓ 是                          ↓ 否
④ 转存新文件                      跳过，不发通知
   ↓
⑤ 立即删除旧 strm
   ↓
⑥ daemon 自动清 115 回收站
   ↓
⑦ 推送【网盘洗版】转存升级 通知
```

### PT 洗版（下载前拦截）

**与原版 MP PT 洗版的区别：**

| 特性 | 原版 MP | 魔改版 |
|------|---------|--------|
| 触发时机 | 下载后对比 | **下载前拦截** |
| 评分门槛 | 无，规则组匹配即下载 | 候选评分 > 现有评分才放行 |
| 低效下载 | ❌ 容易下到提升不大的资源 | ✅ 评分不够直接拦截 |

```
① PT 站点 RSS → 候选种子
         ↓
② MP 规则组评分 + 体积评分
   → 综合评分
         ↓
③ 候选评分 > 现有 strm 评分？
         ↓ 是                      ↓ 否
④ 放行下载                   拦截下载（不发通知）
   ↓
⑤ PT 客户端下载
   ↓
⑥ 入库整理 → 生成 strm
   ↓
⑦ 清理旧 strm 文件
   ↓
⑧ 推送【PT洗版】下载升级 通知
```

### 原生洗版集成（best_version）

插件可自动管理 MP 原生 `best_version`（洗版）开关：

- **自动模式**（`auto_best_version`）：对所有电视剧订阅自动开启 best_version，无需手动操作
- **手动模式**：在插件 UI 中勾选/取消勾选需要洗版的订阅

### 强制重评分

插件 UI 提供 **「强制重评分」** 按钮，一键遍历所有 strm 文件，基于实际 115 文件大小重新计算评分，修正脏数据。

## ⚙️ 插件对 MP 系统的改动

本插件安装后会对 MoviePilot 进行以下自动修改：

### 1. 注入自定义过滤规则

插件启动时通过 MP 的 FilterModule API 注册 3 条自定义正则规则：

| 规则名 | 正则 | 用途 |
|--------|------|------|
| `HDR_VIVID` | `HDR[._ ][Vv]ivid\|菁彩影像\|HDRVivid` | 识别菁彩影像（HDR Vivid）资源 |
| `10BIT` | `10bit\|12bit\|10-bit\|12-bit` | 识别 10bit/12bit 高色深资源 |
| `60FPS` | `60fps\|60帧\|60FPS` | 识别高帧率资源 |

这些规则写入 MP 系统配置，**保存即生效**，无需手动创建。

> ⚠️ v1.6.6+ 默认关闭自动注入，用户可在 UI 中按需开启 `auto_register_rules` 开关。

### 2. 注入 4 套优先级规则组

| 规则组名称 | 包含规则 | 用途 |
|------------|---------|------|
| 电视剧非杜比画质优先 | 排除杜比、优先 4K/HDR/高码率 | 电视剧默认 |
| 电视剧杜比画质优先 | 含杜比、优先 4K/高码率 | 兼容杜比视界的用户 |
| 电影非杜比画质优先 | 排除杜比、优先 4K/HDR/VIVID/高码率 | 纯电影，不要杜比 |
| 电影含杜比画质优先 | 含杜比、优先 4K/高码率 | 需要杜比视界的电影 |

这些规则组注册到 MP 的 SubsribeRuleGroup 系统，在订阅规则中可直接选用。

> ⚠️ v1.6.6+ 默认关闭自动注入，用户可在 UI 中按需开启。

### 3. 设置重命名模板

插件启动时通过 MP 的 `settings.update_setting` API 设置统一的电视剧/电影重命名模板（Jinja2 格式），确保所有文件命名一致。

## ❓ 常见问题

**Q：安装后插件显示「未适配当前版本」？**
A：检查 MP 版本是否 ≥ v2.13.3，或尝试修改 `package.v2.json` 中的 `mp_version` 范围。

**Q：规则组不生效？**
A：检查「自动注册过滤规则」开关是否开启，以及二级分类是否匹配你选的映射。

**Q：洗版不执行？**
A：确认已开启洗版开关，且 cron 时间在允许窗口内。

**Q：需要哪些前置依赖？**
A：必须安装 P115StrmHelper（115 网盘订阅助手）插件，配置好 115 Cookie。

**Q：私有改的插件在线更新会不会覆盖定制？**
A：本仓库只发布魔改版，不会覆盖你的定制。更新前建议备份配置。

**Q：屏蔽态下怎么手动搜 PT？**
A：发送 `/pt_sub_search` 命令（微信/TG），插件会临时放行 60 秒。

**Q：Telegram bot 报 BOT_COMMAND_INVALID？**
A：v1.7.0 已修复，确保命令名只含英文和字母。
